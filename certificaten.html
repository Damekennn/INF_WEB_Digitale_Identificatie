Inleiding

Digitale certificaten zijn electronische documenten die gebruikt worden als digitaal paspoort. Deze worden toegewezen door een zogenoemende Certification Authority (CA). Certificaten identificeren en tegelijkertijd authenticeren de identiteit van een gebruiker op het web. Een getekend certificaat bevat gebruikersinformatie, een bijbehorende public-key en een private-key die alleen de CA weet. Met deze gegevens kan een beveiligde verbinding tussen client en server gemaakt worden.


Public Key Infrastructure


Identificatie is het proces waarbij de gebruiker een gebruikersnaam, email adress of een ander uniek ID naar de webserver toe stuurt. Het maakt hier nog niet uit of deze gebruiker daadwerkelijk de juiste gegevens invoert, soms kan wel worden gecommuniceerd dat er geen account gekoppeld is aan deze gegevens.

Certification Authority


Een certification authority is een organizatie die digitale certificaten bewaard en tekent. Dit is dus een 3e partij, dat door beide andere partijen vertrouwd wordt.


Blockchain


Digitale certificaten worden vaak opgeslagen op een blockchain netwerk, waardoor de data niet meer kan worden aangepast. Het netwerk bestaat uit blokken die hun eigen unieke code en de code van het vorige blok bevatten. Hierdoor zijn alle blokken in een keten aan elkaar verbonden. Deze unieke code is het resultaat van de data uit de digitale certificaten in het SHA256 algoritme. Dit is een complexe berekening, dat veel rekenkracht vereist.
